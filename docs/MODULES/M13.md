# Module M13: Customer Dashboard

This module covers the secure customer authenticated portal, profile management, booking tracking timelines, payment history with failed retries, test drive schedules, and a notification center.

---

# Status

✅ Production Ready

---

# Deliverables

- **APIs**:
  - `GET /api/dashboard/profile` - Fetch customer profile details
  - `PATCH /api/dashboard/profile` - Edit customer address and notifications subscriptions
  - `GET /api/dashboard/bookings` - List authenticated customer bookings
  - `GET /api/dashboard/bookings/:id` - Dynamic tracking timeline for vehicle delivery
  - `GET /api/dashboard/payments` - List customer payments list
  - `GET /api/dashboard/payments/:id` - View payment details, audit logs, and trigger retries
  - `GET /api/dashboard/test-drives` - List customer test drive appointments
  - `GET /api/dashboard/notifications` - Retrieve paginated and searchable customer notification logs
  - `PATCH /api/dashboard/notifications/:id/read` - Set read flag for notification
- **Database changes**:
  - Prisma User schema extended with `address`, `preferredBranchId`, and `communicationPreferences`.
  - Added relation relation `preferredUsers` on `Branch` model.
  - Added `Notification` table to track read/unread customer alerts.
- **Frontend pages**:
  - `/dashboard` - Overview portal welcome layout
  - `/dashboard/profile` - Profile configuration editor
  - `/dashboard/bookings` - My Bookings lists
  - `/dashboard/bookings/[id]` - Graphical vehicle delivery status timeline
  - `/dashboard/payments` - Transaction history table log
  - `/dashboard/payments/[id]` - Invoice details and failed payment retries
  - `/dashboard/test-drives` - Test Drive schedule details
  - `/dashboard/notifications` - Paginated and searchable alert notification logs
  - `/dashboard/settings` - Portal configuration options
