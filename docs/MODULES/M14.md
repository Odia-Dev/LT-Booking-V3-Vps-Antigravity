# Module M14: Delivery Management

Full-stack vehicle handover management module covering backend delivery lifecycle infrastructure, administrative dashboards, customer-facing tracking, multi-channel notifications, and secure document uploads.

---

# Status

✅ Complete — Production Ready

---

# Objectives

- End-to-end vehicle handover management from booking confirmation through physical key delivery.
- Role-based access control for Admins, Sales Executives, and Customers.
- Automated multi-channel notification triggers at each milestone.
- Secure document storage with metadata persistence.

---

# Deliverables

## Task 1 – Delivery Foundation

- **Database Models**: `Delivery`, `DeliveryChecklist`, `DeliveryTimeline`
- **Backend**:
  - `backend/src/modules/delivery/deliveryRepository.ts` – DB queries and transaction controls.
  - `backend/src/modules/delivery/deliveryService.ts` – Status lifecycle validation and checklist enforcement.
- **Prisma**: Schema extended with full relational mappings to `Booking`, `User`, `Vehicle`, `Variant`, `Branch`.

## Task 2 – Delivery APIs

- **Backend**:
  - `backend/src/modules/delivery/deliveryValidation.ts` – Zod schemas for all payloads.
  - `backend/src/modules/delivery/deliveryController.ts` – CRUD, RBAC, filters, pagination.
  - `backend/src/modules/delivery/deliveryRoutes.ts` – Mounted at `/api/deliveries`.
- **Endpoints**:
  - `GET /api/deliveries` – List with search, filters, pagination.
  - `GET /api/deliveries/:id` – Single delivery with full relations.
  - `POST /api/deliveries` – Schedule new handover (Admin only).
  - `PATCH /api/deliveries/:id` – Update metadata (Admin/Executive).
  - `PATCH /api/deliveries/:id/status` – Update delivery status.
  - `PATCH /api/deliveries/:id/checklist` – Update checklist milestones.
  - `DELETE /api/deliveries/:id` – Delete record (Admin only).

## Task 3 – Delivery Checklist

- **10 mandatory gate-pass milestones**:
  1. Payment Cleared
  2. Insurance Issued
  3. RTO Registration Completed
  4. Vehicle PDI Completed
  5. Accessories Installed
  6. Fuel Filled
  7. Cleaning Completed
  8. Documentation Prepared
  9. Delivery Kit Prepared
  10. Customer Orientation Completed
- **Enforcement**: `updateDeliveryStatus` throws a validation error if transitioning to `DELIVERED` with any milestone incomplete.
- **Progress**: Percentage completion tracked per handover.

## Task 4 – Admin Delivery Dashboard

- **Frontend Pages**:
  - `/admin/deliveries` – Paginated list with search, filters (branch/status/executive), checkbox selectors, bulk status/executive updates, and schedule overlay.
  - `/admin/deliveries/[id]` – Detail panel with status timeline, 10-item interactive checklist, status modifier, notes editing, and audit history.
- **Sidebar**: Admin layout navigation extended with Deliveries link.

## Task 5 – Customer Delivery Tracking

- **Frontend Page**: `/dashboard/delivery`
  - 8-step visual milestone tracker (Booking Confirmed → Delivered).
  - Scheduled date/time, branch name, phone, email.
  - Assigned executive display.
  - Multi-booking dropdown selector if customer has multiple deliveries.
- **Sidebar**: Customer dashboard navigation extended with Delivery Tracking link.

## Task 6 – Delivery Notifications

- **Service**: `backend/src/modules/delivery/deliveryNotificationService.ts`
- **6 trigger events**:
  | Event | Trigger Point |
  |---|---|
  | Vehicle Allocated | `createDelivery` |
  | Insurance Issued | Checklist `insuranceIssued` → `true` |
  | RTO Completed | Checklist `rtoCompleted` → `true` |
  | Delivery Scheduled | `updateDelivery` with first `scheduledDate` |
  | Delivery Reminder | Status → `READY` |
  | Vehicle Delivered | Status → `DELIVERED` |
- **Channels**: Email · SMS · WhatsApp (mock dispatchers, production-ready interface).
- **Logging**: `NotificationLog` records (per channel, with `deliveryId`) + `Notification` portal alerts.

## Task 7 – Delivery Documents

- **Database Model**: `DeliveryDocument` – metadata store (type, filename, path, MIME, size).
- **Middleware**: `backend/src/middleware/deliveryUpload.ts` – Multer disk storage under `uploads/delivery/<deliveryId>/`. 10MB limit. PDF/JPEG/PNG/WEBP only.
- **Backend**:
  - `backend/src/modules/delivery/deliveryDocumentController.ts` – Upload, list, delete handlers.
- **Endpoints**:
  - `POST /api/deliveries/:id/documents` – Upload 1-5 files (`multipart/form-data`).
  - `GET /api/deliveries/:id/documents` – List (filter by `?type=`).
  - `DELETE /api/deliveries/:id/documents/:docId` – Delete record + disk file.
- **Document Types**: `INVOICE`, `INSURANCE`, `RC_RECEIPT`, `CHECKLIST_PDF`, `DELIVERY_PHOTO`, `OTHER`
- **Static Serve**: `uploads/` served via Express at `/uploads` with `X-Content-Type-Options: nosniff`.
