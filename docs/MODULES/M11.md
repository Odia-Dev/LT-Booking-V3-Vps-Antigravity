## Technical Specifications

### Booking ID Format
* Pattern: `LT-YYYYMM-XXXXXX`
* Example: `LT-202606-000001`
* Generates sequentially within the current year/month. Safely resolves sequential race conditions by applying retry mechanisms and uniqueness checks.

### Booking Status Lifecycles
* **Initiated**: Booking request initiated, awaiting transaction setup.
* **Payment Pending**: Gateway order generated, customer checkout process started.
* **Payment Successful**: Payment authorized by gateway.
* **Confirmed**: Order reviewed by dealership and confirmed.
* **Vehicle Allocated**: Physical vehicle stock assigned to customer.
* **Delivered**: Vehicle keys handed over to client.
* **Closed**: Order lifecycle complete.
* **Cancelled**: Order aborted (pre-payment or post-confirmation).
* **Refunded**: Funds returned to user.
* **Expired**: Payment window exceeded.

### Payment Statuses
* `PENDING`
* `SUCCESS`
* `FAILED`
* `REFUNDED`
