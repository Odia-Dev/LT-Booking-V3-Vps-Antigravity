# Module M10: Test Drive Management

Outlines the backend and frontend architecture to schedule physical test drive appointments, prevent double bookings, validate showroom slots, manage executives, and send notifications.

---

## Status

🟢 Production Ready (Verified & Build Stable)

---

## Deliverables

- **Backend Repository & Service**:
  - Implemented `testDriveRepository.ts` and `testDriveService.ts` containing customer test drive scheduling CRUD, unique ID generations (`TD-YYYY-XXXX`), filters (status, date, branch), status flow handlers, duplicate prevention rules, and double-booking validations.
- **REST APIs**:
  - Registered routes in `testDriveRoutes.ts` with Zod validation (`testDriveValidation.ts`) and custom error handling.
  - Endpoints: `GET /api/test-drives`, `GET /api/test-drives/:id`, `POST /api/test-drives`, `PUT /api/test-drives/:id`, `PATCH /api/test-drives/:id/status`, `PATCH /api/test-drives/:id/assign`, and `DELETE /api/test-drives/:id`.
  - Mounted public test drive endpoint `POST /api/public/test-drives` for guest user submissions.
- **Frontend Test Drive Booking Experience**:
  - Created `/test-drive` page utilizing search parameters for presets (Vehicle Details CTA and Branch CTA).
  - Integrated components: Vehicle Selector, Variant Selector, Branch Selector, Preferred Date, Preferred Time, Customer Details (Name, Phone, Email, Message, Consent).
  - Automatically creates a User profile and a Lead linking to the scheduled test drive.
  - Persists tracking parameters: UTM, Referrer, Landing Page.
- **Notification Services**:
  - Email confirmation, SMS hooks, WhatsApp templates, and push notifications triggers configured inside the test drive lifecycle.
- **Calendar Services**:
  - Storing Local `CalendarEvent` records for Appointments, Reminders, and Completions.
  - Exposes clean `ICalendarProvider` interfaces prepared for Google Calendar and Microsoft Outlook synchronization.
