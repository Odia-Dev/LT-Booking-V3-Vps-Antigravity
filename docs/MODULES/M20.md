# M20: Guest Checkout + Optional Account Creation

## Status
IMPLEMENTED

## Overview
Replaces mandatory OTP authentication with WooCommerce-style guest checkout.
Customers can now complete bookings and test drives as guests, with an option to create an account during booking. If account creation is requested, a verification email is sent via Resend. The OTP endpoints are deprecated in favor of standard Email/Password authentication.

## Features
* WooCommerce-style guest checkout (No forced login before booking).
* Option to "Create Account" during online booking.
* Guest user details are stored directly on the `Booking` and `TestDrive` records if no account is created.
* Email-based account verification.
* Standard Email & Password authentication for Customers.
* Password Reset flow via secure tokens (valid for 30 minutes).

## Architecture Changes
* `schema.prisma`:
  * `User`: Added `password`, `isVerified`, `verificationToken`, `verificationTokenExpires`, `resetToken`, `resetTokenExpires`.
  * `Booking`: Made `customerId` optional. Added guest fields (`guestName`, `guestEmail`, `guestPhone`, `guestCity`, `guestState`), `financeRequired`, and `exchangeRequired`.
  * `TestDrive`: Made `customerId` optional. Added guest fields (`guestName`, `guestEmail`, `guestPhone`).
* Removed `OtpVerification` model logic in favor of JWT and bcrypt.
* Added `resend` integration in `emailService.ts` for transactional emails.

## Environment Variables Needed
* `RESEND_API_KEY`: API Key for Resend service.
* `FROM_EMAIL`: Sender email address (e.g., `noreply@yourdomain.com`).
* `NEXT_PUBLIC_APP_URL`: Frontend URL for email links.

## Testing
* Covered via validation schema tests and manual flow tests for `createPublicBooking` and `createPublicTestDrive`.
