# Module M08: Branch Management

Outlines the database structures, Zod validations, repositories, services, controllers, and routing mechanisms to list, create, edit, archive, and delete showroom branches.

🟡 In Progress

# Deliverables

- **APIs**: List Branches (`GET /api/branches`), Create Branch (`POST /api/admin/branches`), Edit Branch (`PUT /api/admin/branches/:id`), Delete/Archive Branch (`DELETE /api/admin/branches/:id`).
- **Database changes**: Extended `Branch` model schema with required fields (`code`, `city`, `district`, `state`, `pincode`, coordinates, status, manager phone).
- **Frontend pages**: Showroom selector cards on public landing views.
- **Admin pages**: Branches CMS dashboard panel (`/admin/branches`).
