# Module M08: Branch Management

Outlines the database structures, Zod validations, repositories, services, controllers, and routing mechanisms to list, create, edit, archive, and delete showroom branches.

---

# Status

✅ Production Ready

---

# Deliverables

- **Backend Repository & Service**:
  - Implemented `branchRepository.ts` and `branchService.ts` containing CRUD methods, lookup helpers, search/sort filters, validation rules (phone and email format checks), and status toggles.
- **APIs**:
  - Protected Admin CMS endpoints (`POST /api/admin/branches`, `PUT /api/admin/branches/:id`, `PATCH /api/admin/branches/:id/status`, `DELETE /api/admin/branches/:id`).
  - Dedicated Public listing and detail query endpoints (`GET /api/public/branches`, `GET /api/public/branches/:slug`).
- **Database changes**:
  - Extended the `Branch` schema in the Prisma file to include `salesManager`, `serviceManager`, and `sortOrder`.
- **Frontend pages**:
  - Public location listings (`/branches`) and detailed profile profiles (`/branches/[slug]`) supporting dynamic Schema.org JSON-LD schemas, Open Graph parameters, canonical headers, and coordinates.
  - Branches CMS dashboard panel (`/admin/branches`) with full creation/edit and status toggling forms.
