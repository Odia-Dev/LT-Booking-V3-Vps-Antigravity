# Module M09: Lead Management

Outlines the database schemas, controller layers, public forms, security middleware, and CRM consoles to manage, route, filter, and assign dealership leads.

---

# Status

✅ Production Ready

---

# Deliverables

- **Backend Foundation**:
  - Implemented `leadRepository.ts` and `leadService.ts` containing CRUD methods, filters, pagination, soft delete (`CANCELLED` status), and metadata parsing.
- **REST APIs**:
  - Protected admin routes for CRM console operations (status PATCH, assignment PATCH, bulk updates).
  - Production public endpoint `/api/public/leads` with rate limiter (max 5 requests/minute) and spam keyword filtering.
- **Notification Pipeline**:
  - Configured `notificationService.ts` to trigger customer email notifications, admin alerts, webhook HTTP requests, and WhatsApp draft logs.
- **Frontend Lead Forms**:
  - Built reusable `LeadForm.tsx` component supporting 6 customer enquiry types (General, Call Back, Quote, Brochure, Finance, Exchange) with automatic UTM parameter tracking.
  - Public `/contact` layout supporting tab selection of lead forms.
- **Admin CRM Panel**:
  - Primary list board `/admin/leads` supporting pagination, search, status/type filters, and multi-select bulk operations.
  - Lead details profile `/admin/leads/[id]` displaying campaign metrics, timeline history, and new notes editor.
